# Примеры Chaos-экспериментов (ДЗ 5)

Опора для пункта «3 эксперимента Chaos Engineering». Здесь — структура эксперимента
и **каталог типов** отказов как образцы. Конкретные 3 эксперимента вы формулируете
под свою систему: гипотеза, blast radius и ожидаемый результат — ваши.

---

## Структура любого chaos-эксперимента

Хаос — это не «давайте что-нибудь сломаем», а проверка гипотезы:

1. **Steady state** — что считаем «нормой» (метрика: напр. успешные заказы/мин в пределах X).
2. **Гипотеза** — «при отказе Y система сохранит steady state, потому что есть Z (retry/fallback/реплика)».
3. **Blast radius** — минимальный масштаб (один инстанс, 5% трафика, staging) — ограничиваем урон.
4. **Метод** — что именно ломаем и как.
5. **Что наблюдаем** — какие метрики/алерты подтвердят или опровергнут гипотезу.
6. **Rollback / abort** — условие немедленной остановки эксперимента.

> Хороший эксперимент **может опровергнуть** гипотезу. Если результат известен заранее — это не эксперимент.

## Каталог типов отказов (выберите и адаптируйте 3)

| Тип | Что моделирует | Пример гипотезы для проверки |
|---|---|---|
| **Kill инстанса** | падение пода/ноды сервиса | «трафик переедет на здоровые инстансы без ошибок для пользователя» |
| **Сетевая задержка** | деградация сети/зависимости | «при +300ms к ответу БД сработает таймаут и retry, без каскада» |
| **Отказ зависимости** | недоступен внешний сервис/БД | «при падении сервиса рекомендаций основной поток продолжит работать (graceful degradation)» |
| **Исчерпание ресурса** | CPU/память/диск/пул соединений | «при насыщении пула соединений сработает backpressure, а не полный отказ» |
| **Отказ зоны/региона** | падение AZ | «active-active выдержит потерю зоны в пределах RTO/RPO из задания» |
| **Потеря сообщений/lag брокера** | сбой очереди | «при лаге Kafka обработка наверстает без потери событий (at-least-once)» |
| **Clock skew / зависший узел** | частичный отказ | «таймауты отсекут зависший узел быстрее, чем он накопит очередь» |

```

Steady state:    <метрика нормы>
Гипотеза:        <что выдержит и почему — за счёт какого механизма>
Blast radius:    <масштаб: где и сколько>
Метод:           <что и как ломаем>
Наблюдаем:       <метрики/алерты из вашего списка ДЗ 5>
Rollback:        <когда немедленно прекращаем>
Ожидаемый вывод: <что узнаем независимо от исхода>
```

## Самопроверка

- [ ] 3 эксперимента бьют по **разным** механизмам отказоустойчивости (не три раза «убить инстанс»)
- [ ] У каждого есть **измеримый steady state** и условие **rollback**
- [ ] Гипотезы привязаны к вашим паттернам из задания (circuit breaker, retry, репликация, DR)
- [ ] Blast radius ограничен — эксперимент безопасен
- [ ] Наблюдаемые метрики берутся из вашего же списка RED/USE
