## ДЗ 5. Отказоустойчивость, observability и тестирование

### Цель
спроектировать механизмы отказоустойчивости, наблюдаемости и тестирования распределённой системы для обеспечения её стабильной и безопасной эксплуатации.

### Задание
Для системы, выбранной в ДЗ 2:

**Отказоустойчивость:**
1. Определите RTO и RPO для каждого сервиса (с обоснованием)
2. Спроектируйте репликацию для критичных БД
3. Определите паттерны отказоустойчивости (circuit breaker, retry, timeout, bulkhead)
4. Опишите план DR (что делаем при падении региона)

**Observability:**
5. Составьте список ключевых метрик (15-20 метрик по RED/USE)
6. Опишите 5-7 алертов с порогами и реакцией
7. Определите, что логируем и трейсим

**Тестирование:**
8. Опишите план нагрузочного тестирования (сценарии, метрики успеха)
9. Предложите 3 эксперимента Chaos Engineering

**Безопасность (кратко):**
10. Выберите механизм аутентификации/авторизации
11. Определите, какие данные шифруем

### Референс для выполнения
- Пример плана DR из занятия 26
- Чек-лист метрик RED/USE
- Шаблон описания алертов
- Примеры Chaos-экспериментов

### Формат сдачи
- Таблица RTO/RPO по сервисам
- Схема репликации и DR
- Список метрик и алертов
- План тестирования
- Краткое описание безопасности

### Критерии оценки
1. Адекватные RTO/RPO (не "0 для всего")
2. Корректная схема репликации
3. Осмысленные метрики и алерты (не копипаста)
4. Реалистичный план тестирования

**Статус "Принято"** при выполнении всех критериев.
