## ДЗ 2. Проектирование взаимодействия сервисов

### Цель
Спроектировать взаимодействие сервисов распределённой системы с выбором протоколов, API-контрактов и архитектурных паттернов для обеспечения согласованной работы компонентов.

### Задание
Выберите одну новую систему для работы в ДЗ 2:

| Система | Описание | Ключевые сложности |
|---------|----------|-------------------|
| **Доставка еды** | Аналог Delivery Club | Геолокация, real-time статусы, пиковые нагрузки |
| **Платёжный сервис / кошелёк** | Аналог ЮMoney | Идемпотентность, строгая консистентность, транзакции, антифрод | 
| **Сервис бронирования** | Аналог Booking | Инвентарь, оплата, отмены |

Можно выбрать свою систему, если она соответствует следующим критериям:
- Существует в реальной жизни и имеет определённую бизнес-логику
- Имеет достаточное количество сложностей для проектирования
- Не является слишком простой или тривиальной

Перед выполнением ДЗ с самостоятельно предложенной системой согласуйте с преподавателем, чтобы убедиться, что она соответствует требованиям.

Для выбранной системы:

1. **Декомпозиция** — выделите 6-8 основных сервисов с описанием ответственности
2. **Протоколы** — определите протоколы взаимодействия (REST, gRPC, GraphQL, events)
3. **Схема взаимодействия** — нарисуйте схему с указанием протоколов
4. **API контракты** — опишите 3-4 ключевых API (Swagger/OpenAPI или описание)
5. **Асинхронность** — укажите, где применили бы асинхронное взаимодействие и почему
6. **Паттерны** — какие паттерны применили (Saga, CQRS, etc.) и почему

### Референс для выполнения
- Пример декомпозиции из занятий 1 модуля
- Шаблон описания API
- Таблица сравнения протоколов

### Формат сдачи
- Схема взаимодействия (PNG/PlantUML)
- Описание сервисов и их ответственностей
- API контракты (Markdown или Swagger)
- Обоснование выбора протоколов и паттернов (1-2 страницы)

### Критерии оценки
1. Логичная декомпозиция на сервисы (нет "god service", нет избыточного дробления)
2. Обоснованный выбор протоколов с учётом требований
3. Корректные API-контракты
4. Применены подходящие паттерны

**Статус "Принято"** при выполнении всех критериев.
