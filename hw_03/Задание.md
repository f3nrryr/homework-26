## ДЗ 3. Проектирование хранения данных

### Цель
Спроектировать архитектуру хранения данных распределённой системы с выбором моделей данных, типов баз данных и стратегий масштабирования для обеспечения требуемых характеристик системы.

### Задание
Для системы, выбранной в ДЗ 2:

1. **Модель данных** — определите, какие данные хранит каждый сервис
2. **Выбор БД** — выберите тип БД для каждого сервиса с обоснованием
3. **Шардирование** — спроектируйте схему шардирования для 1-2 ключевых сервисов
4. **Кэширование** — определите стратегию кэширования (что кэшируем, какая стратегия, TTL)
5. **Очереди** — где применили бы брокер сообщений, какой и почему
6. **CAP trade-offs** — как ваше решение соотносится с CAP-теоремой

### Референс для выполнения
- Пример проектирования хранения из занятия 20
- Таблица сравнения БД с критериями выбора
- Шаблон схемы шардирования
- Чек-лист стратегий кэширования

### Формат сдачи
- Схема данных по сервисам (ER-диаграмма или описание)
- Таблица выбора БД с обоснованием
- Схема шардирования с описанием стратегии
- Описание кэширования и очередей

### Критерии оценки
1. Корректный выбор типов БД под задачи (не "PostgreSQL везде")
2. Логичная схема шардирования с обоснованием выбора ключа
3. Продуманная стратегия кэширования (не "кэшируем всё")
4. Понимание CAP trade-offs

**Статус "Принято"** при выполнении всех критериев.
